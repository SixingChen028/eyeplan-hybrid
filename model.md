# Model specification

This document specifies the model implemented in this repository. It describes the decision tree environment, the working-memory dynamics, the value-update dynamics internal to the environment, the observation emitted to the policy, the action space and rewards, and the actor-critic architectures. Training procedures and optimizer details are not part of the model specification.

## Overview

Each episode is a finite tree-search problem embedded in a working-memory-limited cognitive architecture. The tree is a rooted binary decision tree. Nodes have point values. The agent can spend time fixating nodes that are currently active in working memory. Fixating a node reveals local structural and value information around that node, refreshes a local working-memory neighborhood, and updates stored node values. At any time, the agent can terminate search and execute a final move from the root to a terminal node. The final move is evaluated as the expected reward of a stochastic tree traversal determined by the current stored node values.

The environment is internal to the agent. Its state variables represent the contents and decay of working memory, persistent value estimates, and the currently fixated node. Actions are therefore operations over information currently available in working memory, not movements in an external task world.

## Tree distribution

An episode contains `num_nodes = N` nodes, where `N` must be a positive odd integer. Each tree is a full binary tree with `N` nodes and `(N + 1) / 2` leaves.

Tree shapes are generated by the following distribution:

1. Start from a single empty leaf.
2. Repeat `(N - 1) / 2` times:
   - enumerate the leaves in the current tree;
   - choose one leaf uniformly;
   - replace the chosen leaf with an internal node with two empty children;
   - canonicalize left-right ordering so that mirror-equivalent trees are represented by the same shape.
3. Aggregate probabilities of canonical shapes that can be reached through multiple expansion orders.

At reset, one canonical tree shape is sampled from this distribution. Nodes are then assigned breadth-first integer identifiers in the unpermuted template. If `shuffle_nodes` is true, a random permutation of node labels is applied to the root, parent array, and child array. The permutation changes labels but not tree structure.

The environment stores:

- `root_node`: the root node index.
- `child_nodes[i]`: the two children of node `i`, or `-1` entries for terminal nodes.
- `parent_nodes[i]`: the parent of node `i`, or `-1` for the root.

## Point values and path values

Each node has a point value `r_i`. At reset, values are sampled independently and uniformly from `point_set`, except that the root value is set to zero:

```text
r_root = 0
r_i ~ Uniform(point_set), for i != root
```

The default point set is:

```text
{-8, -4, -2, -1, 1, 2, 4, 8}
```

The environment also computes a path-prefix value `g_i` for every node. `g_i` is the sum of point values on the path from the root to the parent of `i`, excluding `i` itself:

```text
g_root = 0
g_i = g_parent(i) + r_parent(i), for i != root
```

For a terminal node `l`, the complete raw path value is:

```text
G_l = g_l + r_l
```

## Environment state

At time `t`, the environment state contains:

- `time_elapsed`: number of actions taken since reset.
- `fixation_node`: currently fixated node.
- `root_node`, `child_nodes`, `parent_nodes`: sampled tree structure.
- `points`: node point values `r_i`.
- `g_values`: path-prefix values `g_i`.
- `q_values`: persistent node-value estimates `Q_i`.
- `n_visits`: number of times each node has been fixated.
- `fixation_recency`: per-node recency trace used for optional observations.
- `activation`: per-node working-memory activation values.
- `rng_key`: random state for stochastic decay, drift, drops, and sampling.

At reset, the tree and point values are sampled, `Q_i = 0`, `n_visits_i = 0`, recency traces are zero, and activations are zero. The reset then performs an initial fixation of the root using the same fixation update used for ordinary node-fixation actions. Thus the first observation is after the root has been fixated once.

## Actions

The action space has `N + 1` discrete actions:

- Actions `0, ..., N - 1`: fixate the corresponding node.
- Action `N`: terminate search and execute the final move.

Only active working-memory nodes may be fixated. Let `A_i` denote activation. At non-final search times, node `i` is a legal fixation action if `A_i > 0`. The terminate action is always legal. At `time_elapsed = t_max - 1`, all fixation actions are masked out, so only terminate remains legal.

The action mask is:

```text
legal_i = (A_i > 0) and (time_elapsed != t_max - 1), for i < N
legal_N = true
```

## Step timing and termination

At each step, `time_elapsed` is incremented by one before the selected action is applied. Recency traces are also decayed before the action-specific update:

```text
R_i <- recency_decay * R_i
```

If the action is a fixation action, the environment performs a node fixation update and returns reward `-cost`. If the action is terminate, the environment returns the scaled expected final move reward and does not perform a fixation update.

An episode ends if either:

```text
action == N
```

or

```text
time_elapsed == t_max
```

Because the action mask permits only terminate at `time_elapsed = t_max - 1`, ordinary policy rollouts terminate by choosing the terminate action before an additional fixation can occur.

## Fixation update

When node `i` is fixated, the environment applies the following operations in order:

1. Set `fixation_node <- i`.
2. Increment `n_visits_i`.
3. Set `fixation_recency_i <- 1`.
4. Update working-memory activation.
5. Update persistent node-value estimates.
6. Corrupt stored Q-values outside working memory.

### Working-memory activation

Before local refresh, every activation decays multiplicatively:

```text
A_j <- clip(wm_decay * A_j, 0, 1)
```

The fixated node, its parent, its children, and the root are then refreshed to full activation:

```text
A_i <- 1
A_parent(i) <- 1, if parent(i) exists
A_child <- 1, for each child of i
A_root <- 1
```

After refresh, each node is stochastically retained in working memory with probability equal to its activation:

```text
keep_j ~ Bernoulli(A_j)
A_j <- A_j if keep_j else 0
```

Nodes refreshed to activation 1 are therefore retained on that step. Nodes with partial activation may be dropped. Nodes with activation 0 are absent from working memory.

## Stored value estimates

The environment maintains a persistent scalar `Q_i` for each node. These values are internal memory variables used by the final move rule and included in observations. They are not parameters of the policy network.

For a node `i`, define its Bellman target:

```text
T_i = r_i + max_{c in children(i)} Q_c
```

For terminal nodes, absent children contribute zero, so `T_i = r_i`. When a working-memory-restricted backup is explicitly used for an ancestor update, inactive children are treated as having `Q = 0` rather than being removed from the maximization.

### Direct update at fixation

Immediately after working-memory activation is updated, the fixated node is updated toward its target:

```text
Q_i <- Q_i + learning_rate * (T_i - Q_i)
```

This direct update uses all stored child Q-values, regardless of child activation.

### Ancestor backup

After updating the fixated node, value updates may propagate up the ancestor chain. The first ancestor update has weight `lamda_backup`; after each updated ancestor, the weight is multiplied by `lamda_backup` again. Backups stop when any of the following conditions holds:

- the current node is the root;
- there is no parent;
- the number of ancestor backup steps has reached `backup_steps`;
- the backup weight is no longer greater than `1e-6`;
- `wm_backup` is true and the next parent to update is not active in working memory.

For an updated ancestor `a`, the step size is:

```text
alpha_a = learning_rate * backup_weight
```

and the update is:

```text
Q_a <- Q_a + alpha_a * (T_a - Q_a)
```

If `wm_backup` is false, the target `T_a` uses all stored child Q-values. If `wm_backup` is true, inactive children contribute `Q = 0` in the target calculation.

## Q-value corruption outside working memory

After the fixation and backup updates, stored Q-values for inactive nodes are corrupted. Active nodes are unchanged by this corruption step.

For inactive node `j`, the stored value is first decayed:

```text
Q_j <- q_decay * Q_j
```

Gaussian drift is then added:

```text
Q_j <- Q_j + epsilon_j
epsilon_j ~ Normal(0, q_drift^2)
```

Finally, the value may be reset to zero:

```text
D_j ~ Bernoulli(q_drop_rate)
Q_j <- 0 if D_j = 1
```

The default parameters set `q_decay = 1`, `q_drift = 0`, and `q_drop_rate = 0`, so inactive Q-values are preserved by default.

## Final move and reward

The terminate action evaluates the final tree traversal from the root using the current stored Q-values. At each internal node `i`, the two children are selected with a softmax over their stored Q-values, mixed with uniform random choice:

```text
p(c | i) = (1 - eps_move) * softmax(beta_move * Q_c) + eps_move / 2
```

where the softmax is over the two children of `i`. Let `E_i` denote the expected raw future point total from node `i`'s children onward under this traversal rule:

```text
E_i = sum_{c in children(i)} p(c | i) * (r_c + E_c)
```

For terminal nodes:

```text
E_i = 0
```

The terminate reward is:

```text
reward = scale_factor * E_root
```

The environment returns this expectation directly. It does not sample or store an actual final root-to-leaf path. The cost term is not subtracted on the terminate action; costs are incurred only on fixation actions.

## Observation

The observation is a tuple of arrays. Most node-indexed fields have length `N`. Optional scalar fields are included depending on environment configuration.

Always included fields:

- `fixation`: one-hot vector for the current fixation node.
- `fixation_point`: scalar point value of the current fixation node.
- `parent`: one-hot vector for the parent of the current fixation node, or all zeros at the root.
- `child`: multi-hot vector for the children of the current fixation node, or all zeros at a terminal node.
- `root`: one-hot vector for the root node.
- `g_values`: path-prefix values for known nodes, with unknown entries set to zero.
- `q_values`: current stored Q-values for all nodes.
- `n_visits`: fixation counts for all nodes, represented as floats.
- `is_terminal`: indicator for terminal nodes that have been fixated at least once.
- `time_elapsed`: scalar elapsed time.

Optional fields:

- `best_open_value`: scalar maximum `g_i` among known, unvisited nodes.
- `best_terminal_value`: scalar maximum complete raw path value `g_i + r_i` among visited terminal nodes.
- `recency`: per-node fixation recency trace.

A node is treated as known if its parent has been visited. The root is also known. This means that after a node is fixated, its children become known open nodes, but their own point values are observed only when they are fixated. The `g_values` field exposes `g_i` for known nodes and zero for unknown nodes.

The `best_open_value` field is:

```text
max_i g_i, over nodes with known parent and n_visits_i = 0
```

If no node satisfies the mask, the value is the minimum path sentinel:

```text
ceil(N / 2) * min(point_set)
```

The `best_terminal_value` field is:

```text
max_i (g_i + r_i), over terminal nodes with n_visits_i > 0
```

If no terminal node has been visited, the same minimum path sentinel is used.

The recency trace is decayed at every step before the action is applied, and the fixated node is then set to one during fixation. Thus `recency_decay = 0` records only the current fixation, while `recency_decay = 1` preserves all previous fixations at value one.

## Actor-critic policy

The model uses an actor-critic network that maps observations to:

- one logit for each node-fixation action;
- one logit for the terminate action;
- one scalar state-value estimate.

Action logits are masked before sampling or greedy action selection. Illegal actions receive the minimum representable floating-point value before the softmax or argmax is applied.

Two architectures are implemented: a flat multilayer perceptron and a shared-node architecture. The shared-node architecture is the default.

## Flat MLP architecture

The flat MLP first concatenates all observation fields in a fixed order:

```text
fixation, fixation_point, parent, child, root, g_values, q_values,
n_visits, is_terminal, [best_open_value], [best_terminal_value],
[recency], time_elapsed
```

Optional fields are included only when enabled. The concatenated vector is passed through two ReLU hidden layers of width `hidden_size`. Separate linear heads produce action logits and the scalar value estimate.

## Shared-node architecture

The shared-node architecture applies the same node encoder to every node. It is permutation equivariant for node logits and permutation invariant for the terminate logit and value estimate, conditional on corresponding permutation of node-indexed observation fields and action masks.

For each node `i`, the network constructs a node feature vector:

```text
x_i = [
    fixation_i,
    parent_i,
    child_i,
    root_i,
    g_i,
    Q_i,
    n_visits_i,
    is_terminal_i,
    recency_i,       # included only if recency observations are enabled
    legal_i
]
```

The `legal_i` feature is the current action-mask entry for node `i`. Without recency, the node feature dimension is 9. With recency, it is 10.

Each node feature vector is passed through a shared two-layer ReLU MLP:

```text
h_i = ReLU(W_node2 * ReLU(W_node1 * x_i + b_node1) + b_node2)
```

A shared linear policy head maps each node embedding to its node-fixation logit:

```text
logit_i = w_node_policy^T h_i + b_node_policy
```

The terminate action and state value are computed from pooled embeddings of legal fixation actions. The model computes:

```text
mean_legal = mean({h_i : legal_i = true})
max_legal = elementwise_max({h_i : legal_i = true})
```

If there are no legal node-fixation actions, the mean uses denominator one and the max pooling output is a zero vector. These cases occur when the timeout mask leaves only the terminate action legal.

The global feature vector is:

```text
z = [
    mean_legal,
    max_legal,
    fixation_point,
    time_elapsed,
    best_open_value,       # included only if enabled
    best_terminal_value    # included only if enabled
]
```

This vector is passed through one ReLU hidden layer:

```text
u = ReLU(W_global * z + b_global)
```

Separate linear heads produce the terminate logit and value estimate:

```text
logit_terminate = w_terminate^T u + b_terminate
V = w_value^T u + b_value
```

The final policy-logit vector is:

```text
[logit_0, ..., logit_{N-1}, logit_terminate]
```

## Default model configuration

The canonical default runtime configuration is:

```text
num_nodes = 15
t_max = 50
scale_factor = 0.125
shuffle_nodes = true
use_recency_obs = true
use_best_open_value_obs = true
use_best_terminal_value_obs = true
wm_backup = true
beta_move = 40.0
eps_move = 0.0
learning_rate = 1.0
lamda_backup = 1.0
backup_steps = 100
wm_decay = 1.0
q_drop_rate = 0.0
q_drift = 0.0
q_decay = 1.0
recency_decay = 0.5
cost = 0.01
point_set = (-8, -4, -2, -1, 1, 2, 4, 8)
network_type = "node_shared"
hidden_size = 64
```
